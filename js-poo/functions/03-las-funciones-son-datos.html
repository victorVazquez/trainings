Las funciones son datos (Functions are data)

En Javascript, las funciones pueden definirse de 2 formas:

Notacion literal
var f = function(){return 1;}

Notacion declarativa
function f(){return 1;}

Asi que las funciones en Javascript son datos, pero un tipo especial de datos con 2 importantes caracteristicas:

  • Contienen código
  • Son ejecutables (pueden ser invocadas)

Funciones anonimas

En Javascript esta bien tener piezas de codigo que se extienden alrededor del programa. Imaginemos que tenemos el siguiente codigo:
"test"; [1,2,3]; undefined; null; 1;
Este codigo puede verse un poco extraño porque no hace nada, pero el codigo es valido y no causa ningun tipo de error.

Ahora, esas piezas anonimas de codigo dispersas alrededor del programa no son utiles, excepto si pasan a ser funciones, en este caso puede haber 2 usos elegantes para ellos:

  • Puedes pasar una funcion anonima como parametro a otra funcion. La función que recibe puede hacer algo util con la funcion que pasas
  • Puedes definir una funcion anonima y ejecutarla enseguida

Veamos esas 2 aplicaciones de las funciones anonimas con mayor detalle

Funciones Callback

Este es un ejemplo de una funcion que acepta 2 funciones como parametros:

function invoke_and_add( a,b ) {
 return a() + b();
}

Ahora, definamos 2 funciones sencillas que solo devuelvan valores codificados:

function one() {
 return 1;
}

function two() {
 return 2;
}

Ahora pasamos esas 2 funciones como parametros de la funcion primera ( B( A ) ):

invoke_and_add(one,two)

O podriamos pasarle 2 funciones anonimas como parametros ( B( function(){return 25;} ) ):

invoke_and_add(function(){return 5;}, function(){return 25;})

Cuando pasamos una funcion A a otra funcion B y B ejecuta A, decimos que A es una funcion callback. Si A no tiene nombre, entonces podemos decir que es una funcion anonima callback.

    Cuando son utiles las funciones callback? Estos son algunos de los beneficios que aportan:
  • Nos permiten pasar funciones sin necesidad de nombrarlas (Lo cual significa que hay menos variables globales)
  • Podemos delegar la responsabilidad a una funcion de llamar a otra funcion (Lo cual significa menos codigo que escribir)
  • Pueden ayudar a mejorar el rendimiento
